Investigating Offline Map Issues

I've analyzed the current implementation. Here's how it's designed to work:

ONLINE MODE
JAWG STREETS (Tiles)

Loads rich vector tiles from api.jawg.io. Caches to jawg-maps-cache.

OFFLINE MODE
LOCAL GEOJSON

Loads 14MB tuzla-map.geojson. Simple roads and paths rendering.

What did you experience?

A

Completely Blank

No map data visible at all, even after waiting.

B

Glitchy / Partially Loaded

Some roads visible, but very slow or missing detail.

C

Stuck on "Online" Map

Still trying to load tiles but showing grey squares.

D

Performance Issues

Map loaded but app became very laggy/unresponsive.

Why it might fail

  • 14MB GeoJSON is too big for some devices
  • Network detection thinks you're online when you're not
  • Path to GeoJSON is incorrect in relative vs absolute
