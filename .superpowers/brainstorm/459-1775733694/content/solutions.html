Solutions: Making Offline Maps Bulletproof

Since both App and Browser failed, we need a solution that works regardless of platform.

1

Active Fallback + Relative Pathing

Instead of relying on navigator.onLine, we'll try to load the online style. If it fails (for any reason), we immediately switch to the local geojson using a relative path that works in both Capacitor and Browser.

2

Optimized GeoJSON + UI Feedback

14MB is heavy for a single frame. I'll check if we can compress the data. Plus, I'll add a "Low Connectivity / Offline" overlay so walkers know the app has switched to "Eco/Map Mode".

3

Force Pre-cache Priority

Modify the Service Worker to treat the map data as "Critical Infrastructure"—ensuring it's fully downloaded and verified the moment the app is first opened online.

My Recommendation

I suggest Approach 1 + 2. Combined, they fix the loading logic, handle the pathing issue, and ensure the UI doesn't look "broken" when it's actually just in offline mode.
